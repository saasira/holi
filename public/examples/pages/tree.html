Tree Variants

Variant 1: component="tree"

Vertical tree, children collapsed by default, click to expand.

Variant 2: role="tree" with single-expand

Only one sibling branch can stay open at each level.

Variant 3: <tree> with lazy children loading

Large datasets can load children on first expand using children-loader + remote endpoint /examples/api/tree?parent=<id>. Uses cache with cache-ttl.

Variant 4: drag-drop hooks

Enable draggable-nodes and optional reorder-handler callback.
